# How to contribute

## Submitting changes

Please send a [GitHub Pull Request](https://github.com/bds-ailab/syn2real/pulls)  with a clear list of what you've done. 
Please follow our coding conventions (below) and make sure all of your commits are atomic (one feature per commit).

Always write a clear log message for your commits. One-line messages are fine for small changes, but bigger changes should look like this:

    $ git commit -m "A brief summary of the commit
    > 
    > A paragraph describing what changed and its impact."

## Testing

Please write unit tests for new code you create, with specifications matching the tests. 
Also add a coverage report in the PR description. We usually use pytest.

Remember that:

- 1 Class = 1 set of tests (usually all the tests related to a unit
- Names are as in [RFC](https://datatracker.ietf.org/doc/html/rfc2119)
- unit MUST/SHOULD/CAN assertion

Useful link: [Pytest](https://docs.pytest.org/en/8.0.x/)

## Coding Conventions

To lint python, we use [Black](https://black.readthedocs.io/en/stable/index.html)
You can install the VS Code extension [Black formatter](https://marketplace.visualstudio.com/items?itemName=ms-python.black-formatter) to use it locally.
